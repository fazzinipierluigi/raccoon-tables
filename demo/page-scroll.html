Page-Scroll Mode (default)

When height is not set, the grid enters page-scroll mode: it grows to its full data height (or page-height when paginated) and the user scrolls the page, not a container. The column header sticks to the viewport. Virtual scroll is still active — only visible rows are rendered.

Pagination works normally in this mode: the body shrinks to pageSize × rowHeight and page changes scroll back to the grid top. Pass height: N to switch to the classic fixed-height / internal-scroll mode.

Content below the grid

This block sits in the normal document flow below the grid. When pagination is off, the grid grows to display all rows and this content is pushed far down the page. When pagination is on, the grid body is only pageSize × rowHeight pixels tall and this content is reachable right below the pagination bar.
